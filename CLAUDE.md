## Overview

Roblox game project using [Rojo](https://rojo.space/) for filesystem-to-Studio syncing. Code is written in [Luau](https://luau.org/) (Roblox's typed Lua variant).

## Toolchain

Tools are managed by **Rokit** (`rokit.toml`) and **Aftman** (`aftman.toml`). Both manage Rojo тАФ Rokit is the primary manager.

Install tools:
```bash
rokit install   # or: aftman install
```

## Common Commands

```bash
# Build the place file from source
rojo build -o "game.rbxlx"

# Start the live-sync server (connect from Roblox Studio via Rojo plugin)
rojo serve
```

## Project Structure

`default.project.json` maps `src/` directories to Roblox service locations:

| Directory | Roblox Location |
|---|---|
| `src/server/` | `ServerScriptService.Server` |
| `src/client/` | `StarterPlayer.StarterPlayerScripts.Client` |
| `src/shared/` | `ReplicatedStorage.Shared` |

- Server scripts run only on the server (`*.server.luau`)
- Client scripts run only on the client (`*.client.luau`)
- Shared modules go in `src/shared/` and are accessed via `ReplicatedStorage.Shared`
- `FilteringEnabled` (FE) is active, so serverтЖФclient communication requires `RemoteEvent` / `RemoteFunction`

## Development Workflow

1. Run `rojo build -o "game.rbxlx"` to generate the place file
2. Open `game.rbxlx` in Roblox Studio
3. Run `rojo serve` to start live syncing тАФ file changes in `src/` reflect immediately in Studio
4. Test in Studio's Play mode; deploy via Roblox's publish workflow
